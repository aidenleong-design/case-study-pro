# Narrative Frameworks

The framework serves the story, not the other way around. First find the story
shape, then pick the structure that carries it. Do not default to the same
structure for every project — variety across a portfolio is itself a signal.

## Step 1: Find the story shape

Ask the user: **"What's the most interesting part of this project?"** The answer
points to a shape:

| Story shape | What carries it | Good frameworks |
|---|---|---|
| A hard design decision | the decision is the climax | Decision Log, Three-Act, Pyramid |
| A craft achievement | the work itself argues | Craft essay (6-part), Before/After/Bridge |
| A research discovery | an insight drives direction | Jobs-to-be-Done, Problem→Solution→Impact |
| Messy process → good outcome | pivots and recovery | Hero's Journey, Pixar, Three-Act |
| Collaboration / alignment | negotiation and trade-offs | Decision Log, STAR |
| Systemic change | before/after at system scale | Before/After/Bridge, Pyramid |

## Step 2: Pick a framework

### 6-Part Default
Overview → Challenge → Process → Solution → Impact → Reflection.
The reliable, scannable structure. Best when the work speaks for itself and you
want range of evidence. ~800–1,500 words.

### STAR
Situation → Task → Action → Result.
Tight and interview-optimized. ~250–400 words. Best for verbal/behavioral
interview prep, not portfolio sites.

### Hero's Journey
Ordinary world → call → trials → transformation → return.
Best for messy-middle stories with real struggle and a clear before/after in the
designer's own thinking. Risk: melodrama — keep it grounded.

### Problem → Solution → Impact
Three roughly equal sections. Clean and businesslike. Best when each leg has
strong, specific evidence.

### Before / After / Bridge
Pain state → new state → the work that connected them. Best for redesigns and
systemic change. Requires honest, annotated comparisons (see structure-guide).

### Pyramid Principle (Minto / BLUF)
Answer first, then three supporting arguments, then detail. Best for executive
readouts and time-poor reviewers. Lead with the outcome.

### Pixar Structure
"Once upon a time… every day… until one day… because of that… until finally."
Best for narrative-forward storytelling (conference talks). Use sparingly.

### Three-Act
Setup → confrontation → resolution. A flexible dramatic spine when there's clear
tension and a turning point.

### Jobs-to-be-Done
Anchored to job statements ("When I ___, I want to ___, so I can ___") and the
behavioral shift the work produced. Best for research-led and product work.

### Decision Log
Chronological key decisions, each with context, options, choice, and trade-off.
Best when judgment under constraint is the headline. Honest and very credible.

## Notes on application

- Match word count and pacing to the framework (STAR is short; a craft essay can
  run longer with visuals).
- Frameworks are blueprints, not cages — bend section order to the story.
- For slide output, see `../templates/slide-template.md`; the standard sequence
  maps loosely to the 6-part default, the growth sequence to Pyramid.
